# Трекер активности на iOS в фоне

## Текущее поведение

- **В foreground (приложение на экране)** — трекер работает: акселерометр через `expo-sensors` обновляет статистику сидения/стояния.
- **В background (свёрнутое приложение)** — iOS не отдаёт события с датчиков в JS. Учёт приостанавливается; после возврата в приложение статистика не искажается (пауза не засчитывается).

## Почему на iOS не работает в фоне

1. **Ограничения iOS** — в фоне система приостанавливает выполнение приложения и не доставляет данные с акселерометра в JavaScript.
2. **expo-sensors** — работает только когда приложение в foreground; в Expo (и в React Native в целом) нет поддержки непрерывного чтения Motion/Accelerometer в background.

## Что можно сделать на iOS

### Вариант 1: Оставить как есть (рекомендуется)

Использовать трекер при открытом приложении. Подсказка в настройках Health уже сообщает: «В фоне учёт приостанавливается».

### Вариант 2: Development build + нативный модуль (сложно)

Чтобы попытаться считать движение в фоне на iOS, нужны:

1. **Development build** (не Expo Go), например:
   ```bash
   npx expo install expo-dev-client
   npx expo prebuild
   ```
2. **Нативный iOS-модуль** (Swift), который:
   - использует `CMMotionManager` (Core Motion);
   - при включённом трекинге запускает обновления движения;
   - пишет агрегированные данные (например, «сижу»/«стою» и время) в общее хранилище (UserDefaults/файл) или шлёт в JS через bridge при следующем выходе в foreground.

Важно: даже с нативным кодом iOS может приостанавливать приложение в фоне, и тогда Core Motion перестанет вызывать обработчики. Гарантированно «живое» выполнение в фоне Apple даёт в основном для:

- **Location** (режим `location` в UIBackgroundModes) — для навигации/карт; использование только ради того, чтобы «держать приложение живым», без реального использования геолокации может привести к отклонению в App Store.
- **Audio** — воспроизведение (в т.ч. тихий звук); использование только для удержания в фоне тоже может вызвать вопросы при ревью.

Для трекера активности Apple не предоставляет отдельного разрешения на непрерывный фоновый доступ к Motion.

### Вариант 3: Фоновые задачи (expo-background-task)

Можно включить периодические фоновые задачи (например, раз в 15–30 минут). Они **не** дают непрерывного потока данных с акселерометра, только короткий запуск кода по расписанию. Для подсчёта сидения/стояния в реальном времени этого недостаточно, но можно использовать для других фоновых задач (синхронизация, уведомления и т.п.).

## Итог

| Платформа | Foreground | Background |
|----------|------------|------------|
| iOS (текущее приложение) | ✅ Работает | ❌ Учёт приостанавливается |
| iOS (с нативным модулем) | ✅ | ⚠️ Только при условии, что iOS не приостановит процесс; гарантий нет |
| Android (текущее приложение) | ✅ | ❌ Аналогично без Foreground Service |
| Android (с Foreground Service) | ✅ | ✅ Возможно нативным кодом |

Для iOS в рамках текущего Expo-проекта корректный вариант — считать трекер работающим только при открытом приложении и не закладываться на фоновый учёт без отдельной нативной реализации и рисков со стороны политики Apple.
