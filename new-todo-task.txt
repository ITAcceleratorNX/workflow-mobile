ТЗ — Этап 3
Команды и назначение задачи

1. Цель

Реализовать базовый командный функционал:

создание команды;
добавление участников;
назначение задачи на команду;
назначение задачи на исполнителя.

2. Создание команды

Где находится
Во вкладке “Входящие” справа должна быть отдельная кнопка / зона, открывающая боковую панель.

Что находится в боковой панели
список команд;
кнопка “Создать команду”;
в будущем здесь может размещаться статистика выполненных задач.

3. Новая команда

При создании команды указываются:

Название команды
Руководитель команды
Участники команды

Логика
команда создаётся через боковую панель;
при создании можно выбрать несколько участников;
руководитель команды задаётся отдельным полем.



4. Участники команды

Логика
в команду можно добавить несколько человек;
участники команды отображаются внутри команды;
при необходимости участника можно убрать из команды.

Руководитель команды
является отдельной сущностью внутри команды;
не заменяет список участников;
задаётся при создании команды.

5. Назначение задачи

Где работает

Назначение команды и исполнителя должно быть доступно:
при создании задачи;
при редактировании уже созданной задачи.

Где доступно

Назначение должно работать в любом разделе:

Входящие
Сегодня
Предстоящие

6. Поля внутри задачи

В задаче должны быть два отдельных поля / действия:

Назначить команду
Назначить исполнителя

Логика

Оба поля работают независимо друг от друга.

7. Назначить команду

Поведение

пользователь нажимает “Назначить команду”;
открывается список доступных команд;
пользователь выбирает нужную команду;
задача становится командной.

Результат

Если у задачи выбрана команда, задача считается задачей команды.
То есть, сколько человек в команде – у каждого высвечивается задача.

8. Назначить исполнителя

Поведение
пользователь нажимает “Назначить исполнителя”;
открывается список пользователей;
пользователь выбирает нужного исполнителя;
задача закрепляется за этим человеком.

9. Варианты назначения задачи

Вариант 1

Задача без команды и без исполнителя - обычная задача.

Вариант 2

Задача только с исполнителем.
Задача назначена на конкретного человека.

Вариант 3

Задача только с командой.
Командная задача без конкретного исполнителя.

Вариант 4

Задача с командой и исполнителем.
Задача относится к команде, но отвечает за неё конкретный человек.

10. Логика командной задачи

Если выбрана только команда
задача видна всем участникам команды;
задача считается общей командной задачей.
Если выбраны команда и исполнитель
задача относится к команде;
задача также закреплена за конкретным исполнителем.

11. Редактирование уже созданной задачи
Логика

У уже созданной задачи пользователь должен иметь возможность:

назначить команду;
изменить команду;
удалить команду;
назначить исполнителя;
изменить исполнителя;
удалить исполнителя.

12. Основные сценарии

Сценарий 1

Пользователь открывает боковую панель во Входящих - нажимает “Создать команду” - указывает название, руководителя и участников - команда сохраняется.

Сценарий 2

Пользователь создаёт задачу - нажимает “Назначить команду” - выбирает команду - задача становится командной.

Сценарий 3

Пользователь создаёт задачу - нажимает “Назначить исполнителя” - выбирает человека - задача назначается на исполнителя.

Сценарий 4

Пользователь создаёт задачу - выбирает и команду, и исполнителя - задача относится к команде и закрепляется за выбранным человеком.

Сценарий 5

Пользователь редактирует уже созданную задачу - добавляет или меняет команду / исполнителя.
