# Структура проекта workflow-mobile

## app/
Экраны и маршруты (Expo Router). Структура как в проде (kcell-service-front): маршрут = папка, экран = index.tsx.
- **_layout.tsx** — корневой layout, Stack (вкладки + экраны авторизации).
- **index.tsx** — корневой экран (редирект на логин или табы).
- **(tabs)/** — таб-навигация.
  - **_layout.tsx** — конфигурация нижней панели (одна вкладка «Главная»).
  - **index.tsx** — главный экран с кнопкой «Регистрация».
- **login/index.tsx** — экран входа (телефон + пароль).
- **register/index.tsx** — экран регистрации (3 шага: форма → SMS → пароль).
- **reset-password/index.tsx** — восстановление пароля (телефон → SMS → новый пароль).
- **privacy/index.tsx** — политика конфиденциальности (RU/EN).

## components/
- **themed-text.tsx**, **themed-view.tsx** — базовые компоненты с поддержкой светлой/тёмной темы.
- **haptic-tab.tsx** — кнопка вкладки с тактильной отдачей (используется в табах).
- **ui/** — переиспользуемые UI-компоненты:
  - **button.tsx** — кнопка (primary / secondary / ghost).
  - **text-input.tsx** — поле ввода с подписью и опциональным показом пароля.
  - **select.tsx** — выбор из списка (Modal + список).
  - **otp-input.tsx** — ввод 6-значного кода.
  - **success-modal.tsx** — модальное окно успеха.
  - **icon-symbol.tsx** — иконки (SF Symbols на iOS, Material Icons на Android).
  - **index.ts** — реэкспорт всех UI-компонентов.

## lib/
API, конфигурация и утилиты.
- **config.ts** — конфиг приложения (базовый URL API из env).
- **api.ts** — запросы к бэкенду: офисы, категории услуг, создание заявки на регистрацию.
- **auth.ts** — отправка и проверка SMS-кода верификации.
- **registration.ts** — константы и утилиты для регистрации (роли, формат телефона, начальное состояние формы).
- **index.ts** — реэкспорт из lib для удобного импорта.

## constants/
- **theme.ts** — цвета (light/dark), шрифты, акцентные цвета для кнопок и полей.

## hooks/
- **use-color-scheme.ts** — текущая тема (light/dark).
- **use-theme-color.ts** — цвет из темы по имени.

## assets/
Изображения и статика (при необходимости).
